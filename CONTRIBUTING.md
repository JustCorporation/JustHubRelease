# Contribuição

Este repositório mantém o distribuição pública do JustHub. Atualmente ele é mantido por uma única pessoa, com apoio de ferramentas de IA.

## Fluxo

1. Leia o `README.md` e o `AGENTS.md`, quando existir.
2. Crie uma branch específica para a tarefa.
3. Faça uma alteração pequena e focada.
4. Execute os testes e o build aplicáveis.
5. Revise o diff completo.
6. Abra um Pull Request descrevendo a mudança, validações e riscos.
7. Faça merge somente depois de revisar o resultado.

## Não faça

- trabalhar diretamente na `main`;
- misturar tarefas diferentes no mesmo Pull Request;
- adicionar dependências sem justificar;
- alterar produção, releases, secrets ou infraestrutura sem autorização;
- apagar branches, arquivos ou históricos sem verificar o impacto.

## Uso de IA

A IA pode implementar a tarefa, mas o resultado deve ser revisado. O Pull Request deve informar:

- o que foi alterado;
- quais arquivos foram modificados;
- quais testes foram executados;
- limitações ou riscos conhecidos.
